One of the primary reasons for pulling tcom-core into another project is to make use of the base templates that provide the header, footer and overall styles. There are a few options in regards to how tcom-core may be used.

  • Static Integration
  • Custom Compiled
  • Grunt Integration

Static Integration

A static integration is suitable for cases where the parent project simply needs the base styles and markup as the starting point for a new development, eg. a stand alone Java application. In cases like this, the project will not be setting up it's own grunt build and consequently cannot make use of the source .less, .hbs files etc.

Suitable for:

  • Application builds without a grunt build
  • Standalone builds hosted outside of telstra.com.au AEM

Pros:

  • No compiling necessary, simply a case of copying the required source files into a project and using the base markup template
  • Works as is

Cons:

  • Updating is a manual process
  • Can't optimise the build by omitting certain aspects.

How to

  1. Clone the dist branch (refer to Getting Started).
  2. Copy the contents of tcom-core/dist folder to a new location as the basis for a new project.
  3. Files that are safe to delete:
    /*.html - except for dist-template.html
    /local-only folder
    /js/*.debug.js files
    /css/*.map files
  4. Use the dist-template.html file as the basis for your markup. All page content should be placed within the area commented in the source.

Custom Compiled

A custom compiled integration is the same as the above Static Integration except that you wish to optimise the build and compile the code yourself.

  1. Clone the master branch or required tag (refer to Getting Started).
  2. Refer to Getting Started on how to setup for compiling.
  3. Run grunt or grunt build to build and then refer to Static Integration How To.

Grunt Integration

For projects that are running a frontend grunt build, this integration method provides the most flexibility and allows for full usage of the Page Template options.

Suitable for:

  • Projects with grunt builds such as Angular application builds

Pros:

  • Allows for full use of template features
  • Flexible, build can be customised for needs if required
  • Easy to update if using git subtrees

Cons:

  • More effort to setup

How to

  1. Add tcom-core as a subtree within your project, refer to Git Subtrees.
  2. Setup grunt build to run tcom-core build as first step within project build. The following custom task will spawn a separate grunt task to do so. Include this within the project's Gruntfile.js and call the run-core-grunt task as normal (customise the cwd path if required). // Custom grunt runner for tcom-core grunt.registerTask('run-core-grunt', function () { var done = this.async(); console.log('Building tcom-core...'); grunt.util.spawn({ grunt: true, args: ['build'], opts: { cwd: 'src/tcom-core' } }, function (err, result, code) { console.log(result.stdout); done(); }); });
  3. Setup the project's assemble build to correctly refer to tcom-core files. Note the comments inlined below for which options are required and what they do. module.exports = { options: { layoutdir: './src/tcom-core/src/html/layouts', // allows usage of responsive.hbs base template data: [ // required for template './src/tcom-core/src/html/data/*.{json, yml}' ], partials: [ // required for template './src/tcom-core/src/html/partials/**/*.hbs', './src/global/html/partials/**/*.hbs', ], helpers: [ // required for template './src/tcom-core/src/html/helpers/*.js' ], prettify: { indent: 2 }, responsive: true, // sets all pages built to responsive corePath: '/tcom-core', // path to tcom-core subtree, for css/js references headerBefore: 'aem-global-css', footerAfter: 'aem-global-js', primaryNav: 'mega-menu-documentation', title: 'tcom-aem-multi' }, ... build targets }
  4. Ensure that all contents of tcom-core/dist are available on the corePath specified above, so that the generated css folder is available within your corePath folder. This can be accomplished with a copy task.

For an example of how tcom-core can be integrated, refer to the tcom-aem-multi repository.
