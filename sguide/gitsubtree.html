subtrees are similar to the concept of submodules, whereby a child repository (tcom-core) is pulled into a parent (tcom-aem-multi). However, unlike submodules - which store a SHA commit reference - subtrees take a literal snapshot of the files and store them within the parent. Imagine it as effectively copy and pasting the child into the parent repository.

For practical purposes, this can be treated as just another folder. The benefits of the subtrees are:

  • Easier to understand, there is a lot of misunderstanding regarding how submodules work.
  • Easier to manage, as the subtree is simply a file folder, making modifications within it is the same as modifying any other file. Changes on multiple branches don't cause reference conflicts and changes can be backported to tcom-core as required in a separately managed process.
  • Isolated, as the files are snapshotted into the parent - changes within the folder are isolated to tcom-aem-multi and do no affect tcom-core. However, as the subtree is still linked to tcom-core it is possible to push and pull updates to tcom-core.

More can be found regarding subtrees here and here.

Refer to Integration for more information.

Adding a subtree

A remote called tcom-core is assumed, refer to the Subtree Tips section.

To add tcom-core as a subtree within your repository, the following command can be used from your repository root:

C:\repository-root > git subtree add --prefix=[path] [remote] [branch/tag] --squash
Parameter Description
--prefix=[path] The path where tcom-core should be added as a subtree. Eg src/tcom-core.
[remote] Remote branch url or the name of a local remote repository. If you have added the remote as per above, it will be called tcom-core.
[branch/tag] Name of the branch or tag to pull.
--squash Squashes all commit messages into a single commit, keeps multiple commits from tcom-core from polluting the parent repository.

The command would be the following for adding a subtree to src/tcom-core from the tcom-core remote and master branch.

C:\repository-root > git subtree add --prefix=src/tcom-core tcom-core master --squash

Updating a subtree

A remote called tcom-core is assumed, refer to the Subtree Tips section.

Changes can be pulled from the original subtree repository with the following. Note the paramters are the same as adding, except we've replaced add with pull.

C:\repository-root > git subtree pull --prefix=src/tcom-core tcom-core master --squash

Subtree Tips

Some useful notes with regards to working with a subtree.

Add as remote

It is useful to setup the reference to the tcom-core repository as a remote within your local tcom-aem-multi repository. This will allow you to refer to the repository by name instead of by url reference, which is required when issuing git subtree commands. Run the following to be able to setup a remote called tcom-core:

> git remote add tcom-core ssh://git@git02.ae.sda.corp.telstra.com/tel/tcom-core.git

NB: This example uses ssh, however your access to stash might be setup as https. If that is the case, you can retrieve the correct url by going to tcom-core on stash and clicking on 'Clone'.

To verify it has been setup, running git remote -v should show the following:

> git remote -v origin ssh://git@git02.ae.sda.corp.telstra.com/tel/tcom-aem-multi.git (fetch) origin ssh://git@git02.ae.sda.corp.telstra.com/tel/tcom-aem-multi.git (push) tcom-core ssh://git@git02.ae.sda.corp.telstra.com/tel/tcom-core.git (fetch) tcom-core ssh://git@git02.ae.sda.corp.telstra.com/tel/tcom-core.git (push)

Issue commands from root

git subtree commands only function from the repository root. In this case, that means from the tcom-aem-multi directory.

For example, a git subtree pull command.

C:\tcom-aem-multi > git subtree pull --prefix=tcom-aem-frontend/src/tcom-core tcom-core master --squash
