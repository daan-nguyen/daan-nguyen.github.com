Icon Example

Generate the required icon code using the size and colour controls below. Please note, default size will always inherit the size value from the parent element the icon is inserted to. For example, it will adhere to the size of the current line of text.

Hover states can be added via .td-icon-hover and a lock icon can also be added by embedding a lock icon within the element itself (should only be used with .td-icon-md).

Icons can be used in a variety of elements. Use the default <i> for an inline element, <div> for block elements and <a href="..."> for links. Text can also be added under an icon by using the <p> element.

Size: Default | xs | sm | md | lg | xl

Colour: Default (grey) | Theme Light | Theme Dark | White

Element: i | div | a

Creation Process

For the best results, we use a web application called Icomoon to generate the font files from the source svg files. Below are instructions on replicating the process for adding a new svg icon to the font set.

  1. Open up the Icomoon app projects page.
  2. Select Import Project and browse to the selection.json file found in tcom-aem-frontend/src/global/html/data/selection.json and select it.
  3. A new project will appear, load it and you should be presented with a page of selected icons under the heading telstra-icons.
  4. To add a new icon to this set, select the hamburger icon at the far right end of the telstra-icons set title and select Import to set.
  5. Select the .svg file for the new icon you are adding. It will be added to the start of the set as unselected.
  6. Either select the unselected icon manually or use the select all option under the burger menu, then press the Font button at the bottom.
  7. This screen should tell you the total number of glyphs you are exporting, select Download to download the generated font file.

Copying into tcom-aem-frontend

  1. Once you've downloaded Icomoon generated file, unzip it.
  2. Copy all font files in the fonts directory (.eot, .woff, .ttf and .svg) to tcom-aem-frontend/src/fonts, overwriting any old td-original-icons.* files.
  3. Copy selection.json file from base directory to tcom-aem-frontend/src/html/data/ and overwrite any old selection.json file.
  4. Once the above files have been copied in, the standard Grunt builds will generate the updated stylesheets and demo pages from the selection.json file.
  5. Run build task grunt build-icons to generate less file for new icons.
