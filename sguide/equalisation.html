Content Equalisation

Equalisation runs at two levels, the primary type which will be covered here and a secondary type on .separators and .col-borders that is run passively outside of any developer intervention.

Primary equalisation will take a .row with the attribute data-equalise and attempt to equalise all elements within a specific structure (below). It will take every child within .col .section...

<div class="row" data-equalise> <div class="col"><!-- one of your column --> <div class="col-wrapper"> <div class="parbase section"> <p><!-- section 1st child --> <img src="" alt=""> </p> <h4></h4><!-- section 2nd child --> </div> <div class="parbase section"> <p></p><!-- section 3nd child --> <h4></h4><!-- section 4nd child --> </div> </div> </div> <!-- ...more columns --> </div>

... and equalise it across all .col columns within the .row to equivilent child elements.

<div class="row" data-equalise> <div class="col"> <div class="col-wrapper"> <div class="parbase section"> <p style="height:50px;"></p><!-- index 0 --> <h4 style="height:80px;"></h4><!-- index 1 --> </div> </div> </div> <div class="col"> <div class="col-wrapper"> <div class="parbase section"> <p style="height:50px;"></p><!-- index 0 --> <h4 style="height:80px;"></h4><!-- index 1 --> </div> </div> </div> </div>

Separator Equalisation

Separator (secondary) equalisation runs regardless of any tagged content on .row elements that make use of .separators or .col-borders. It has been optimised so that if a .row has the data-equalise attribute it will equalise columns as part of the standard process, otherwise a standalone equalisation will run for .row elements without data-equalise set.

The example below shows the same content in two tabs. The first tab has the data-equalise attribute set whilst the second one doesn't. In both cases the .col-borders are equalised despite the uneven content.

Disabling Equalisation

Not all elements require equalisation, individual elements within a row that has data-equalise enabled can be excluded by adding the data-equalise-none attribute to the element.

<div data-equalise-none></div>

The example below excludes .callout as the callout is positioned absolutely and has a set size, equalisation causes issues with it.

<div class="row" data-equalise> <div class="col"> <div class="col-wrapper"> <div class="parbase section"> <p></p><!-- ok --> <h4></h4><!-- ok --> <p></p><!-- ok --> <a href=""></a><!-- ok --> <div class="callout" data-equalise-none></div><!-- sorry, not you --> </div> </div> </div> </div>

Hidden content equalisation

A variety of components hide their content initiall, due to this equalisation will not run on them. Instead, various events will trigger equalisation to occur when these components show their contents. The components in question are; accordions, tabs & switches and lightboxes.

Below an example of these components in action, nothing different needs to be done to enable the equalisation on these components. Simply use the data-equalise attribute on .row elements as outlined above.

Equalisation in Javascript

Equalisation is running first on .hero-row and then on body.

// Equalisation is running on the .hero-row to simulate a fast page rendering tcom.equalise($('.hero-row'), true); // Equalisation is running on the body once the page is loaded. $(window).load(function() { tcom.equalise($('body'), true); });

You can run Equalisation on a specific element with tcom.equalise( $target, _fixMerge ).

// $target = dom element parent; // _fixMerge = bolean, handle the merge margin or not; tcom.equalise( $target, _fixMerge );
